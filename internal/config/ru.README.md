# internal/config

Загрузка локальной YAML-конфигурации и наложение значений из etcd.

`config.go` — определяет все структуры конфигурации, загружает и валидирует `config.yaml`. Ключевые особенности:
- `RolesMap.UnmarshalYAML` извлекает зарезервированный ключ `defaults` и применяет его как базу к каждой записи роли (поля с нулевым значением наследуются из defaults).
- `ActorCommand` принимает как строку YAML (`"script.sh arg"`), так и список.
- `Merge` возвращает shallow-копию `*Config` с override значениями из etcd там, где они не пусты; node-локальные поля (endpoints, listen, node_id, actors_base_dir) всегда берутся из локальной конфигурации.
- `ResolveActorPath` объединяет относительный путь к актору с `Node.ActorsBaseDir`.

`watcher.go` — `Watcher` следит за префиксом `config/{cluster_id}` в etcd и перестраивает сводный снапшот конфигурации при каждом изменении. В настоящее время только логирует изменение; живое распространение на запущенных агентов — задача будущего.
