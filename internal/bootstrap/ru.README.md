# internal/bootstrap

Однократное заполнение etcd при запуске агента.

`Bootstrapper.Ensure` записывает начальное состояние кластера в etcd, используя `TryPutIfAbsent` для общих ключей (конфигурация кластера, определения ролей, конфигурация групп управления, desired state) — побеждает первый запущенный узел. Собственная конфигурация узла (`config/nodes/{id}`) использует обычный `Put`, чтобы изменения имени и членства в локальном файле конфигурации применялись при перезапуске.

Начальный документ `desired`, записываемый для каждой management group: `state=passive, managed=false`. Это означает, что свежеподключённый узел пассивен и находится вне зоны управления контроллера до тех пор, пока оператор явно не включит управление. Первый узел, записавший ключ, побеждает (`TryPutIfAbsent`); последующие агенты не перезаписывают уже управляемую группу.

Bootstrap не создаёт ключи actual/health ролей — они являются runtime-ключами, привязанными к сессионному lease, и не должны существовать в etcd между перезапусками агента.
