# internal/store

Кэш состояния etcd в памяти, организованный как дерево с индексацией по пути.

`StateStore` хранит все ключи etcd под корнем кластера в виде дерева `TreeNode` (каждый сегмент пути — узел дерева). Является единственным источником истины для контроллера, воркеров ролей и обработчика API — ни один компонент не читает из etcd напрямую после запуска.

Ключевые операции:
- `LoadSnapshot` — заменяет всё дерево данными из массового чтения etcd; переводит хранилище в состояние ready.
- `Apply` — применяет одно событие put/delete из потока watch etcd.
- `Snapshot` — возвращает deep-копию дерева, безопасную для конкурентного использования обработчиком API.
- `Get` / `Prefix` / `ListChildren` — точечные и диапазонные запросы без обращений к etcd.
- `Notify` — возвращает буферизованный канал, получающий сигнал после каждого `Apply` или `LoadSnapshot`; используется обработчиком SSE потока.
