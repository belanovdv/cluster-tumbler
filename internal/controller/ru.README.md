# internal/controller

Цикл согласования, работающий только на лидере. Запускается, пока данный узел удерживает lease лидерства в etcd.

`Reconcile` итерирует каждую группу управления:
1. Проверяет отсутствие ожидаемых ключей состояния ролей (обнаружение потери агента).
2. Агрегирует actual/health отдельных ролей в actual/health уровня группы.
3. Записывает actual и health группы обратно в etcd только при изменении.

`applyPriorityPolicy` реализует автоматический failover: находит доступную группу с наименьшим номером приоритета и устанавливает ей `active`; всем остальным — `passive`.

Не реализовано: auto failback, anti-flapping, quorum policy, обработка очереди команд (чтение `commands/{id}`, записанных через API, выполнение запрошенного изменения desired state и архивирование в `commands_history/{id}`).
